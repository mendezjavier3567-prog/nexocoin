# Security Policy

## Supported Versions

Nexocoin is under active development.  
Security reviews and updates apply to the latest version of the protocol as published in this repository.

Only the most recent stable implementation is considered supported for security evaluation.

---

## Reporting a Vulnerability

The Nexocoin project takes security seriously.

If you discover a potential security vulnerability, please report it responsibly using **private disclosure**.

### How to Report

- Do **not** open a public GitHub issue for security-related reports.
- Contact the core development team privately.

### Recommended Information to Include

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact on the network or users
- Any suggested mitigation, if available

---

## Response Process

All security reports are reviewed by the core team.

- Valid reports will be acknowledged
- Issues will be investigated and prioritized
- Fixes will be implemented following a conservative and security-first approach
- Public disclosure will occur only after mitigation is available

---

## Security Philosophy

Nexocoin follows a conservative, correctness-oriented security philosophy:

- Minimal attack surface at the base layer
- Predictable behavior over experimental features
- Explicit design goals prioritizing long-term network safety
- Gradual upgrades with backward compatibility when possible

Security is treated as a continuous process, not a one-time event.
